For security reasons, we have discontinued the support for 3DES Cipher. After this discontinuation of support for 3DES cipher, machines running on Windows XP, that do not support any more recent and secure ciphers than 3DES, will not be able to access the Sign In and My Account pages.

If you're a Windows XP user, we strongly recommend upgrading your OS.

EventType

DESCRIPTION

An object containing constant properties that represent the event types used for monitoring statuses of a webOS Signage device. Monitoring targets include temperature, fan, lamp, screen and signal status of the signage device.

SINCE
SCAP v1.0
Platform Compatibility
All

CONSTANT PROPERTIES

Name Value Description
Signage.EventType.CURRENT_TEMPERATURE "CURRENT_TEMPERATURE" The event is for temperature monitoring.
Signage.EventType.FAN_STATUS "FAN_STATUS" The event is for fan monitoring.
Signage.EventType.LAMP_STATUS "LAMP_STATUS" The event is for lamp monitoring.
Signage.EventType.SCREEN_STATUS "SCREEN_STATUS" The event is for screen monitoring.
Signage.EventType.SIGNAL_STATUS "SIGNAL_STATUS" The event is for signal monitoring.

REMARK

An event type is used in the event callback function in Signage.registerSystemMonitor() method. The event handler registered by the Signage.registerSystemMonitor() method receives a monitoring data object when an event is fired. Each event type has a different properties in its object.

Event Property Property Type Description
CURRENT_TEMPERATURE Temperature Number Current temperature of webOS Signage device.
FAN_STATUS Status String Fan status
  • "ok": Good
  • "fault": There is either a hardware or software problem with the fan.
  • "na": The signage device has no fan.
LAMP_STATUS Status String Lamp status
  • "ok": Good
  • "fault": There is either a hardware or a software problem with the lamp.
  • "na": The signage device has no lamp.
SCREEN_STATUS Status String Screen status
  • "ok" : Good
  • "ng" : No good. There is a problem with the screen.
SIGNAL_STATUS Status String Signal status
  • "no_signal": There is no signal.
  • "signal_available": Signal is available

IsmMethod

DESCRIPTION

An object containing constant properties that represent ISM (Image Sticking Minimization) methods; these ISM methods are provided to prevent burn-in.

SINCE
SCAP v1.0
Platform Compatibility
All

CONSTANT PROPERTIES

Name Value Description
Signage.IsmMethod.COLORWASH "COLORWASH"

The Color Wash option removes permanent images from the screen by moving a grid of colored blocks from the bottom of the screen to the top, every second.
Signage.IsmMethod.INVERSION "INVERSION" The Inversion option inverts the panel color of the screen.
Signage.IsmMethod.NORMAL "NORMAL" The Normal option indicates using no ISM feature.
Signage.IsmMethod.ORBITER "ORBITER" The Orbiter option makes the screen image move every two minutes to prevent the same pixels from being used for a prolonged period of time.
Signage.IsmMethod.USER_IMAGE "USERIMAGE" The User Image option displays the desired images using a USB flash drive.
Signage.IsmMethod.USER_VIDEO "USERVIDEO" The User Video option displays the desired video.
Signage.IsmMethod.WASHING_BAR "WASHINGBAR" The Washing Bar option displays a bar on the screen to make the image move smoothly.
Signage.IsmMethod.WHITEWASH "WHITEWASH" The White Wash option removes permanent images from the screen by filling the screen with solid white

registerRS232CEventListener

DESCRIPTION

Registers an event listener to monitor the data received from the RS232C port. The RS232C connection from an external device to a signage device is a prerequisite for communication. If an external device sends the RS232C command to a signage device according to the pre-defined protocol format, an SI app can receive the data from the external device. 

Suppose that an external device wants to send "dummy data" as data to a signage device. The external device sends the RS232C command, "mediabox_dummy data_E6", which follows the protocol format and the RS232 port configuration. The signage device receives the data from a serial port and checks it with checksum. Then, the signage device transfers the data, "dummy data" to the SI app.

RS232C Port Configuration:

- Baud Rate: 9600 bps
- Data Length: 8 bits
- Parity Bits: None
- Stop Bits: 1 bit
- Flow Control (Handshaking): None

 

Serial Protocol Format: 

mediabox_[DATA]_[CHECKSUM]

  • mediabox: The reserved prefix.
  • [DATA]: The data to be sent in the range from 1 to 100 bytes.
  • [CHECKSUM]: The additional 2 bytes in hexadecimal added at the end of the data, after summing all 2-byte values. See the following figure to learn how to make a checksum.
