changeLogoImage

DESCRIPTION

Changes the logo image that is displayed at the time of booting. The default image is the LG logo. The new logo image can be loaded from the local storage, USB flash drive, or remote server. Specify the file location as a URI, as guided below.

URI FORMAT

  • Using a double-dot-slash, ../, in a pathname is NOT ALLOWED for referring to files in local storage and USB storage.
  • No directories or files beyond the base path should be included in the file path.
  • The maximum length allowed for a file URI is 255, except for remote server URI.
  • To access a file in an external storage (USB flash drive or SD card), the external storage must be inserted to the signage device.
  • To access a file in a remote server, use HTTP GET protocol for downloading a file.
Type URI Scheme
Local Storage file://internal/[FILE_PATH]
USB Flash Drive file://usb:[INDEX]/[FILE_PATH]
SD Card file://sdcard:[INDEX]/ [FILE_PATH]
Remote Server

http://[DOMAIN]/[FILE_PATH]

https://[DOMAIN]/[FILE_PATH]
SINCE
SCAP v1.2
Platform Compatibility
All

SYNTAX

Storage.changeLogoImage(successCallback, errorCallback, options)

PARAMETERS

Property Type Description Required
uri String The URI of the new logo image. Required

RETURNS

None

REMARK

The logo image has the following restrictions:

  • Format: JPG or BMP
  • Maximum resolution: Full HD resolution (1920 X 1080)
  • Maximum file size: 8 MB

copyFile

DESCRIPTION

Copies a file from the given source location to the given target location. Available source locations are; remote server, USB flash drive, SD card and local storage. Specify the file or directory as a URI, as guided below.

URI FORMAT

  • Using a double-dot-slash, ../, in a pathname is NOT ALLOWED for referring to files in local storage and USB storage.
  • No directories or files beyond the base path should be included in the file path.
  • The maximum length allowed for a file URI is 255, except for remote server URI.
  • To access a file in an external storage (USB flash drive or SD card), the external storage must be inserted to the signage device.
  • To access a file in a remote server, use HTTP GET protocol for downloading a file.
Type URI Scheme
Local Storage file://internal/[FILE_PATH]
USB Flash Drive file://usb:[INDEX]/[FILE_PATH]
SD Card file://sdcard:[INDEX]/ [FILE_PATH]
Remote Server (HTTP)

http://[DOMAIN]/[FILE_PATH]

https://[DOMAIN]/[FILE_PATH]
Remote Server (FTP)

ftp://user:password@domain:port/[FILE_PATH]

sftp://user:password@domain:port/[FILE_PATH]

ftps://user:password@domain:port/[FILE_PATH]
SINCE
SCAP v1.0
Platform Compatibility
All

SYNTAX

Storage.copyFile(successCallback, errorCallback, options)

PARAMETERS

Property Type Description Required
source String The URI of the file to be copied. Required
destination String The URI of the target location to copy the source file to. If the given target location does not exist, this method automatically creates the given path. Required
ftpOption Object Contains options for copying a file over FTP/SFTP connection. Optional
ftpOption.secure String

Contains modes for FTPS connection.

  • 'implicit': FTPS implicit mode (default)
  • 'explicit': FTPS explicit mode
 Optional
ftpOption.secureOptions Object Contains options for secure FTP connection. Optional
ftpOption.secureOptions.privateKey String The private key for key-based or host-based user authentication. The key is in OpenSSH format. Optional
ftpOption.secureOptions.passphrase String Used for encrypted private key. Optional
ftpOption.connTimeout Number

The timeout period for connection establishment.

  • Unit: Millisecond.
  • Default: 10000 ms.
 Optional
ftpOption.pasvTimeout Number

The timeout period for PASV data connection establishment.

  • Unit: Millisecond.
  • Default: 10000 ms.
 Optional
ftpOption.keepalive Number

The time frequency to send a dummy command (NOOP) to keep a connection alive.

  • Unit: Millisecond.
  • Default: 10000 ms.
 Optional
httpOption Object

Contains options for copying a file over HTTP/HTTPS connection.

For more information about HTTP status code, see Status Code Definitions.

 Optional
httpOption.maxRedirection Number The maximum number of allowed redirections for copying a file over HTTP/HTTPS connection.
An HTTP request will be redirected if the HTTP status code was 301, 302, 303 or 307 and a valid location was found in the HTTP response header. If the source file is not a remote file, this option is ignored.
  • 0: Do not allow redirection. (Default)
  • 1–5: Allow redirections as given.
 Optional
httpOption.headers Object The header, in [name: value] pairs, of an HTTP request for a remote file request. This header data will be cascaded during redirection. The maximum header data size is 1 KB. Optional
httpOption.timeout Number The time out period after which the underlying TCP connection gets disconnected. To keep the connection on without time out, set the value to 0, which is the default value.
N.B. If the server side has a set timeout that is shorter than the value set for this property, copying a file may timeout before the time set for this property is passed.		 Optional

RETURNS

None

REMARK

The copied file is accessible by an app in local storage or in USB flash drive, from the ./content directory of local storage or USB flash drive.

  • The file copied to the local storage is accessible using the following URL.
    http://127.0.0.1:9080/[FILE_PATH]
  • The file copied to a USB flash drive is accessible using the following URL.
    http://127.0.0.1:9080/external/usb/[USB_PORT]/[FILE_PATH]
  • The file copied to an SD card is accessible using the following URL.
    http://127.0.0.1:9080/external/sdcard/[SD_CARD_SLOT]/[FILE_PATH]

If the DNS server value is unset on webOS Signage, accessing the local server with the hostname localhost may take longer than expected to load the resource. Use the local server IP, 127.0.0.1 instead.

upgradeApplication

DESCRIPTION

Upgrades an app in local storage or USB flash drive with the new version of the app from a remote server. The new version of the app will be launched at the next booting of the signage device and at the same time, the previous version of the app will be deleted.

The information for the remote server, from which to download the new version of the app, is configurable on the server settings. The URL of the new app depends on the server settings, as explained below.

Fully Qualified Domain Name (FQDN) is ON

Any URL that starts with an http:// or https://, and ends with zip or ipk is acceptable.

Fully Qualified Domain Name (FQDN) is OFF

Secure Connection is ON: The URL shall be in the following format and start with an https://

https://[SERVER_IP]:[SERVER_PORT]/procentric/scap/application/scap_app.zip

https://[SERVER_IP]:[SERVER_PORT]/application/com.lg.app.signage.ipk

Secure Connection is OFF: The URL shall be in the following format and start with an http://

http://[SERVER_IP]:[SERVER_PORT]/procentric/scap/application/scap_app.zip

http://[SERVER_IP]:[SERVER_PORT]/application/com.lg.app.signage.ipk

SINCE
SCAP v1.0
Platform Compatibility
All

Should this method be called on an application that manages and controls signage devices, prompt for an authentication (e.g. administration permission) before calling the method. This is a recommendation to prevent unintended changes.

SYNTAX

Storage.upgradeApplication(successCallback, errorCallback, options)

PARAMETERS

Property Type Description Required
to String The target storage to store the new app. Required
recovery Boolean Indicates whether to use the recovery mode or not.
  • true: Enable the recovery mode. The recovery mode recovers the old app if the upgrade fails.
  • false: Disable the recovery mode. This option will remove an old app to obtain free space in storage. This option is useful for replacing a relatively large sized app.
 Optional
type String

Indicates the type of app this method is handling.

 Optional

RETURNS

None

REMARK

The size of the new scap_app.zip (zip file itself and the extracted files) or com.lg.app.signage.ipk shall not exceed the free space in the storage. When an app is being upgraded, all of the following files exist at the same time:

  • The old app.
  • The new app downloaded.
  • Content data stored by the Storage.copyFile() in the ./content directory.
  • New app extracted from the downloaded scap_app.zip

Thus check the free space by calling the Storage.getStorageInfo() before upgrading an app.

An app should not contain a directory named content, under the app's root directory. This directory name is reserved for the Storage.copyFile() method.
